Unity notes

  • Disabling a gameobject will NOT stop InvokeRepeating, but WILL stop coroutines. http://answers.unity3d.com/questions/458780/if-you-disable-a-gameobject-does-an-invokerepeatin.html

  • Unity supports hot code reloading, but this can be very difficult to maintain in a working state for projects of any serious complexity. It relies on being able to retain state through a process of serialization, full game reload, and deserialization. Anything that is not linked into the Unity serialization system, such as static variable state, will be lost. If you're making a multiplayer game in particular you might as well forget about it.

  • Unity serialization (of the ISerializationCallbackReceiver variety) is an editor-only thing. It is used for exposing data in the editor and for saving and loading prefabs, but cannot be used to e.g. create save games at runtime.

  • Unity's collision detection system has 31 collision layers that are defined pre-runtime. This can be quite limiting for certain kinds of behavior (for example, preventing self-collision between a variable number of entities). Physics.IgnoreCollision goes some way to solving this.

  • Colliders are abstract components that don't do much by themselves unless they are associated with a Rigidbody. A single Rigidbody will make use of all the Colliders attached to the game object and any children of that object, allowing composite structures composed of multiple Colliders to behave as a single structure physics-wise. However, you cannot have nested Rigidbodies, as they will interact with each other in strange ways.
